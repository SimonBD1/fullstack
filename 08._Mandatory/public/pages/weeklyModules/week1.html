Uge 1

Fullstack

I første uge af Fullstack blev vi introduceret til faget. Denne introduktion var blandt andet omfanget af node.js, kode konventioner og basal javascript.
Vi gennemgik også "Rest API design", hvor vi lærte om opstillingen af ens endpoints.

Node.js

I lidt flere detaljer er Node.js et javascript runtime envoriemnt der tillader at udviklere at skrive server-side kode i javascript, og håndtere asynkrone funktioner. Man starter sin apps op med node ved at skrive ' node "filnavn" ' i terminalen.

Express

express er et web framework til node.js som gør det simpelt at lave app/api udvilking. Dens framework har metoder til mange ting, vi bruger det til foreksempel at håndtere routing og https request.

Rest konventioner

En af de konventioner vi gennemgik var at navngive vores URL's rigtigt. Et url skal være sigende for hvad man kan finde på siden, og ikke have udsagnsord som "get" eller "all". En anden god ting at have i mente er at til andre HTTP metoder man gør brug af, skal den følge samme URL, dog med et paramtre som eksempelvis "ID"

JavaScript

En vigtig detalje i JavaScript, som måske ikke altid er åbenlys, er, hvordan sproget konverterer datatyper. Dette kan føre til uventede resultater, som illustreret i eksemplet nedenfor, hvor det kan være svært at forudsige outputtet på forhånd. Man ville tro at i begge eksempler at der vil bruges samme metoder. Men i det første eksempel vil outputtet blive konverteret til string "55". Og i 2. eksempel til et number, 4. For at undgå forvirring kan man selvfølgelig gøre brug af indbyggede funktioner som "parseInt", "parseFloat", "Number" eller "String" for at overtage stryringen igen.
